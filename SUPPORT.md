# Support

Thank you for using GenixBit Agents.

## Before requesting help

1. Read the main `README.md` and the relevant integration documentation.
2. Confirm that you are using the latest default branch or latest published release.
3. Search existing issues and discussions for the same problem.
4. Reproduce the problem with a minimal, non-sensitive example.
5. Remove API keys, tokens, customer data, private repository details, local usernames, and internal paths from logs.

## Where to ask

### Bug reports

Open a GitHub issue when a documented feature does not work as expected. Include:

- Operating system and version
- AI coding tool or runtime and version
- Installation method
- Agent, script, or integration involved
- Exact reproduction steps
- Expected and actual behaviour
- Sanitised logs or screenshots

### Feature requests and new agents

Use GitHub Discussions for broad proposals, new integrations, architectural changes, or ideas that may affect multiple files. Focused additions or improvements to a single agent may be submitted through a pull request following `CONTRIBUTING.md`.

### Security issues

Do not open a public issue. Follow `SECURITY.md` and use a private GitHub security advisory or email **info@genixbit.com**.

### Commercial and implementation support

For product engineering, custom AI agents, integrations, SaaS development, web or mobile applications, cloud architecture, and business automation:

- Global website: https://www.genixbit.com/
- UK website: https://www.genixbit.co.uk/
- General enquiries: info@genixbit.com
- Business enquiries: sales@genixbit.com

## Support boundaries

Community support does not include access to private GenixBit repositories, client code, credentials, customer information, proprietary datasets, or internal infrastructure.

The maintainers cannot guarantee compatibility with every fork, unofficial package, user-modified agent, third-party model, editor extension, or external service. Reports should clearly distinguish project-controlled behaviour from third-party behaviour.
